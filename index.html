Overview

CoffeeScript on left, compiled JavaScript on right

What is it?

  • CoffeeScript is a language that compiles into JavaScript.
  • Inspired by Ruby and Python
  • Made as an effort to enhance JavaScript's brevity and readability.
  • The golden rule of coffeescript: “IT’S JUST JAVASCRIPT”.

History

  • CoffeeScript is developed by Jeremy Ashkenas. It was first committed in Git On December 13, 2009.
  • Originally the compiler of the CoffeeScript was written in Ruby language.
  • In March 2010, the CoffeeScript compiler was replaced; this time instead of Ruby, they used CoffeeScript itself.
  • Also in 2010, CoffeeScript 1.0 was released and it was one of the most wanted projects of the Git hub.

What are the advantages?

  • Easily understandable − CoffeeScript syntax is simple compared to JavaScript. Codes are more clean, clear, and easily understandable.
  • Less Code, More Functionality − For a huge code in JavaScript, we need comparatively less number of lines of CoffeeScript.
  • Class-based Inheritance - JavaScript does not have classes. CoffeeScript allows for the creation of classes and inheritance. In addition to this, it also provides instance and static properties.
  • Avoids problematic symbols − There is no need to use semicolons and parentheses in CoffeeScript. Instead of curly braces, we can use white spaces to differentiate the block codes like functions, loops, etc.
  • Extensive library support − In CoffeeScript, we can use the libraries of JavaScript and vice versa. Therefore, we have access to a rich set of libraries while working with CoffeeScript.

What are the disadvantages?

  • Debugging is pain − There isn't yet any way to match compiled JS lines to the original CoffeeScript (though the Firefox folks have promised that this is coming).
  • It's not as well-known CoffeeScript has become popular in a short amount of time, but it's unlikely that it'll have as vast as a community as JavaScript.

Getting started

To globally install CoffeeScript, you can use npm (node package manager)
          

  npm install --global coffeescript
To locally install CoffeeScript in a project, you can use npm again
          

  npm install --save-dev coffeescript
To watch for changes in the .coffee file and recompile into js whenever a change occurs,
          

  coffee -watch --compile test.coffee
To get the output from running the compiled JS,
          

  coffee test.coffee
You can see here for more details on usage of CoffeeScript in the command line.

Declaring variables

Declaring variables are less tedious in CoffeeScript. In JavaScript, you have to initialize variables similar to that in Java. In CoffeeScript, however, you can directly give a variable and value. 

              
  physics_c_grade = 72
  that_smells = "like a qaf post"

  # You can also do
  physics_c_grade = 72; that_smells = "like a qaf post"

  console.log physics_c_grade
  console.log that_smells





              
            
              
  var physics_c_grade, that_smells;
  physics_c_grade = 72;

  that_smells = "like a qaf post";

  // You can also do
  physics_c_grade = 72;

  that_smells = "like a qaf post";

  console.log(physics_c_grade);

  console.log(that_smells);
Output
              
  72
  like a qaf post

Functions

Functions in CoffeeScript are also simplified. There are no curly braces, parenthesis, or semicolons necessary. Instead, the code is indented with spaces. 

              
  escalator_works = () ->
    false
  console.log escalator_works();


  is_it_hot = (num) ->
    if num > 0
      "thats hot"
    else
      "its rewind time"

  console.log is_it_hot(-1);





              
            
              
  var escalator_works, is_it_hot;

  escalator_works = function() {
    return false;
  };

  console.log(escalator_works());

  is_it_hot = function(num) {
    if (num > 0) {
      return "thats hot";
    } else {
      return "its rewind time";
    }
  };

  console.log(is_it_hot(-1));
Output
              
  false
  its rewind time

Arrays

Accessing arrays and making subarrays can be done with brackets instead of calling a separate function. This can be seen below. 

              
  physics_test_grades = [97,88,75,49,34,22,1]

  # inclusive range
  console.log physics_test_grades[0..2]

  # exclusive range
  console.log physics_test_grades[0...2]


              
            
              
  var physics_test_grades;

  physics_test_grades = [97, 88, 75, 49, 34, 22, 1];

  // inclusive range
  console.log(physics_test_grades.slice(0, 3));

  // exclusive range
  console.log(physics_test_grades.slice(0, 2));
Output
              
  [ 97, 88, 75 ]
  [ 97, 88 ]

Accessing objects in CoffeeScript and in Javascript are the same.

  #Using comprehension
  kowalski_analysis = (thing) ->
    for year, info of sing_shows
      "#{year} had a " + thing + " of #{info[thing]}"

  console.log kowalski_analysis("top_crew")







              
            
              
  console.log(sing_shows['sophfrosh']['score']['pre_deduction']);

  //Using comprehension
  kowalski_analysis = function(thing) {
    var info, results, year;
    results = [];
    for (year in sing_shows) {
      info = sing_shows[year];
      results.push(`${year} had a ` + thing + ` of ${info[thing]}`);

    return results;
  };

  console.log(kowalski_analysis("top_crew"));

Operators

One of the main advantages of CoffeeScript is that it has many operators that is already familiar to us because they are just English words! Words like “isnt” is the equivalent to “!==“ in JavaScript. This makes coding multiple logic operators easier for the coder and reader.

Loops

Looping is made easier in CoffeeScript. Although looping is still used in the same way, the code in CoffeeScript is much more clean and concise compared to the same code in JavaScript. 

              
  # While Loop
  console.log "Starting While Loop"
  count = 0
  while count < 10
     console.log "Current Count : " + count
     count++;

  # For Loop
  for worst_sing_group,grade in ['SophFrosh', 'Junior']
     console.log "hey hey! ho ho! "+ worst_sing_group + " SING just lost the show."

  #Another For Loop
  values = (num for num in [1..10] by 2)
  console.log values















              
            
              
  // While Loop
  var count, grade, i, len, num, ref, values, worst_sing_group;

  console.log("Starting While Loop");

  count = 0;

  while (count < 10) {
    console.log("Current Count : " + count);
    count++;
  }

  ref = ['SophFrosh', 'Junior'];
  for (grade = i = 0, len = ref.length; i < len; grade = ++i) {
    worst_sing_group = ref[grade];
    console.log("hey hey! ho ho! " + worst_sing_group + " SING just lost the show.");
  }

  //Another For Loop
  values = (function() {
    var j, results;
    results = [];
    for (num = j = 1; j <= 10; num = j += 2) {
      results.push(num);
    }
    return results;
  })();

  console.log(values);

Other Information

Answers to questions from class

Q: Can you access arrays backwards or with negative indexing?
A: Sort of, you can used the command .reverse() to reverse the array.
As an example: 
          
  physics_test_grades = [97,88,75,49,34,22,1]
  console.log physics_test_grades[0..2]
outputs 
          

  [97,88,75]
While 
          

  console.log physics_test_grades[0..2].reverse()
outputs 
          

  [75,88,97]
As for the negative indexing, CoffeeScript doesn’t support negative indexing.

Q: Is CoffeeScript used more to facilitate back-end or used more for front-end?
A: CoffeeScript is mainly a back-end programming language. Especially when it comes to functions, CoffeeScript greatly reduces the number of lines of code needed to perform the same function in JavaScript. Another reason why CoffeeScript is mainly used for back-end is due to its class-based object system, allowing for the creation of classes and inheritance, which is not allowed in JavaScript. Also because browsers do not support CoffeeScript natively, it should be used as back-end.

Q: Can CoffeeScript be used to access DOM elements?
A: Yes. You can use document to access the DOM like in JavaScript or use a library like jquery. The golden rule of CoffeeScript is that “it’s just JavaScript”, so DOM elements can be accessed.
